Ask Halo architecture — compute decision per component

Component Type Lambda viable Fargate viable Critical constraint
Query Router Synchronous Yes Only if always-on 29-sec API GW timeout. Fargate cold start (15–30s) = immediate 504.
Chunk / Embed / Index Async batch Yes (≤15 min) Yes No SLA pressure. Lambda 15-min timeout sufficient for current data volume.
AgentCore (routing logic) Synchronous Yes (embedded) Extra network hop Tokenisation + threshold logic. Negligible compute. Separate Fargate task adds unnecessary latency.
Feedback endpoint Async write Yes Yes (if already running) ~20K writes/month. Trivial compute.
Suggested Qs Synchronous Yes Only if always-on DynamoDB lookup. Sub-100ms. Cold Fargate start unacceptable for UI.
Due Diligence reports (Phase 2) Long-running No (>15 min limit) Yes 30–60 min generation. Exceeds Lambda timeout. Natural Fargate workload.
Real-time event processor (Phase 2) Stateful / 24/7 No (stateless) Yes Persistent WebSocket or streaming connections. Lambda cannot hold state across invocations.
